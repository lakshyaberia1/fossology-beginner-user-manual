# FOSSology Beginner User Manual

## What this repository is

This repository contains a **beginner-friendly user manual for FOSSology**, written for users with **zero prior knowledge**.

The goal of this manual is to help first-time users go from:

- Cloning FOSSology from GitHub  
- Running it using Docker  
- Understanding the web interface  
- Performing their first successful scan  
- Handling common beginner problems  

This manual is based on **real onboarding experience** and focuses on areas where beginners usually get confused.

---

## Who this manual is for

This manual is intended for:

- First-time FOSSology users  
- Students learning open-source license compliance  
- New contributors who want system-level understanding  
- Users confused by Docker, uploads vs scans, jobs, or UI navigation  

This manual is **not intended for advanced administrators or LDAP-only enterprise setups**.

---

## What this manual covers

This manual explains:

- Installing and running FOSSology using Docker  
- Logging in with default credentials  
- Understanding the navigation bar (Home, Browse, Upload, Jobs, etc.)  
- Uploading source code correctly  
- Selecting and running scanning agents  
- Monitoring jobs and understanding failures  
- Viewing and interpreting license scan results  
- Restarting and troubleshooting common problems  
- Real-world and enterprise use cases of FOSSology  

---

## How to read this manual (important)

For best results, read the files **in this order**:

1. `installation.md`  
2. `navigation-bar.md`  
3. `upload-and-scan.md`  
4. `restart-and-troubleshooting.md`  
5. `deep-uses.md`  

Each file focuses on **one responsibility only**, to keep learning simple and structured.
